29 ## Project goals
30 
31 - Demonstrate a hash table in Fortran.
32 - Use standard Fortran 2008.
33 - Single file implementation.
34 - No dependency.
35 - No preprocessing.
36 - Not design a generic container, `dictionary_m` just stores strings. It is of course
37  possible to replace the "value" entry in the type `entry_t` for a single-datatype
38  dictionary.
39 
40 
41 The hash table is based on plain allocatable arrays and the base data is stored in `(len=:),
42 allocatable` character variables. Buckets are extended arbitrarily by reallocation, thus
43 collisions will slow down this implementation with respect to others using better data
44 structures.
45 
46 The goal is not to beat other implementations performance-wise but to provide the
47 convenience for Fortran programmers to store configuration settings, or other auxiliary data
48 easily.

51 ## Installation
52 
53 `dictionary_m` consists of a single Fortran file. You can just drop `src/dictionary_m.f90`
54 in your Fortran project.
55 
56 The code requires Fortran 2008 support. For gfortran, I have tested version 6.3 and 7.2. For
57 Intel Fortran, I have tested version 16.0.3.
